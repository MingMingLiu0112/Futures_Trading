# 缠论线段检测算法学习笔记

## 核心概念

### 笔（Bi）
- 至少5根K线，有明显高低点
- 相邻笔之间有重叠包含关系需合并

### 线段（Segment/XD）
- 线段由笔构成
- UP线段方向向上，包含若干UP笔和DOWN笔（回调笔）
- DOWN线段方向向下，包含若干DOWN笔和UP笔（反弹笔）

### 特征序列（EigenFX）
- UP线段的特征序列 = 构成该线段的**下降笔**序列
- DOWN线段的特征序列 = 构成该线段的**上升笔**序列
- 特征序列用来判断线段是否结束

---

## 线段破坏判断（chan.py算法）

### 三元素分型法
特征序列中前两根笔构成第一二元素，第三根笔触发判断：

```
UP线段（下降笔序列）寻找顶分型破坏：
  - e1是第一条下降笔
  - e2是第二条下降笔  
  - e3是第三条下降笔加入时触发判断
  - 顶分型成立条件：e3.low < e2.low（e3创新低）
  - 顶分型成立 → UP线段结束
```

### 破坏的三种情况（`can_be_end`返回值）
1. **True**：正常破坏，线段确认结束
2. **False**：破坏失败，线段延续（需继续等待）
3. **None**：找到尾部仍未破坏，但特征序列有缺口（gap），需用反向笔处理

### 缺口破坏（`find_revert_fx`）
当特征序列本身不连续（有缺口）时，需要在反向寻找破坏：
- 在反向笔序列中同样用三元素分型法判断
- 找到反向破坏 → 原线段结束

## chan.py关键逻辑

### `treat_fx_eigen`流程
```
1. can_be_end(bi_lst) 检查分型是否有效
2. True/None → add_new_seg（新增线段）
3. False → 从分型位置重新 cal_seg_sure
```

### `is_sure`标志
- 线段笔数 ≥ 3 → is_sure = True（确认线段）
- 线段笔数 < 3 → is_sure = False（虚段，不参与后续分析）

### `actual_break`（实际破坏）
- 防止"假破坏"：特征序列顶分型形成但无实际破坏
- e3加入后，检查e3是否对e2有"实际突破"
- e2是下降笔但e2.next.next没创新低 → 假破坏，线段延续

## 我之前算法的错误

**错误**：认为"3笔同向 → 线段结束"，比高低点就结束

**正确理解（chan.py）**：
- 不是3笔，而是"3个特征序列元素"构成顶/底分型
- 分型成立后，还要检查"实际破坏"（actual_break）
- 即使分型成立，若无实际破坏，线段延续
- 破坏失败（False）时，从分型位置重新搜索
