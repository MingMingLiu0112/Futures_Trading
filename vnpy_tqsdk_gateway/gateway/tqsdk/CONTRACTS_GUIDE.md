# 天勤网关合约配置文件使用说明

## 概述

`tqsdk_contracts.json` 是天勤网关的合约配置文件，用于定义支持的期货和期权合约。该文件采用 JSON 格式，包含每个合约的基本信息。

## 文件位置

```
vnpy/gateway/tqsdk/tqsdk_contracts.json
```

## 配置文件结构

配置文件是一个 JSON 数组，每个元素代表一个合约：

```json
[
    {
        "symbol": "合约代码",
        "exchange": "交易所代码",
        "name": "合约名称",
        "size": 合约乘数,
        "price_tick": 最小价格变动,
        "min_volume": 最小委托量
    },
    ...
]
```

## 字段说明

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `symbol` | string | 是 | 合约代码，必须符合交易所命名规则 | "IF2605", "v2605C5500" |
| `exchange` | string | 是 | 交易所代码，必须使用 vnpy 定义的交易所枚举值 | "CFFEX", "DCE", "SHFE", "CZCE", "INE" |
| `name` | string | 是 | 合约名称，用于显示 | "沪深300指数期货", "PVC期权看涨" |
| `size` | number | 是 | 合约乘数，每手合约对应的标的数量 | 300, 5, 10 |
| `price_tick` | number | 是 | 最小价格变动单位 | 0.2, 5, 1 |
| `min_volume` | number | 是 | 最小委托量（手数） | 1 |

## 交易所代码

| 代码 | 交易所 | 说明 |
|------|--------|------|
| `CFFEX` | 中国金融期货交易所 | 股指期货、国债期货 |
| `DCE` | 大连商品交易所 | 商品期货、商品期权 |
| `SHFE` | 上海期货交易所 | 商品期货、商品期权 |
| `CZCE` | 郑州商品交易所 | 商品期货、商品期权 |
| `INE` | 上海国际能源交易中心 | 原油期货 |

## 合约类型

### 1. 期货合约

期货合约的 `symbol` 字段遵循交易所的标准命名规则：

#### 股指期货（CFFEX）
```
IF2605  - 沪深300指数期货
IH2605  - 上证50指数期货
IM2605  - 中证1000指数期货
```

#### 商品期货（DCE/SHFE/CZCE/INE）
```
rb2605  - 螺纹钢期货（SHFE）
m2605   - 豆粕期货（DCE）
v2605   - PVC期货（DCE）
SR605   - 白糖期货（CZCE）
SC2605  - 原油期货（INE）
```

**命名规则**：
- **DCE/SHFE**：`品种代码 + 年月`（如 `rb2605`）
- **CZCE**：`品种代码 + 月年`（如 `SR605`，年份省略20）

### 2. 期权合约

期权合约的 `symbol` 字段包含期权类型和行权价信息，网关会自动解析期权字段。

#### 股指期权（CFFEX）
```
格式: IO2605-C-4500
- IO: 期权产品代码（沪深300）
- 2605: 到期月份（2026年5月）
- C: 期权类型（C=看涨，P=看跌）
- 4500: 行权价
```

**示例**：
```
IO2605-C-4500  - 沪深300股指期权看涨，行权价4500
IO2605-P-4500  - 沪深300股指期权看跌，行权价4500
HO2605-C-2500  - 上证50股指期权看涨，行权价2500
MO2605-P-5800  - 中证1000股指期权看跌，行权价5800
```

#### 商品期权（DCE/SHFE）
```
格式: v2605C6000
- v: 品种代码（PVC）
- 2605: 到期月份（2026年5月）
- C: 期权类型（C=看涨，P=看跌）
- 6000: 行权价
```

**示例**：
```
v2605C6000  - PVC期权看涨，行权价6000
v2605P6000  - PVC期权看跌，行权价6000
cu2605C65000  - 铜期权看涨，行权价65000
m2605P3000   - 豆粕期权看跌，行权价3000
```

#### 商品期权（CZCE）
```
格式: SR605C6000
- SR: 品种代码（白糖）
- 605: 到期月份（2026年5月，年份省略20）
- C: 期权类型（C=看涨，P=看跌）
- 6000: 行权价
```

**示例**：
```
SR605C6000  - 白糖期权看涨，行权价6000
SR605P6000  - 白糖期权看跌，行权价6000
CF605C15000  - 棉花期权看涨，行权价15000
```

## 手动添加合约步骤

### 步骤 1：确定合约信息

在添加合约之前，需要确认以下信息：

1. **合约代码**：从交易所官网或交易软件获取
2. **交易所**：合约所属的交易所
3. **合约乘数**：每手合约对应的标的数量
4. **最小价格变动**：价格变动的最小单位
5. **最小委托量**：最少可委托的手数（通常为1）

### 步骤 2：添加期货合约

以添加 **PVC期货 v2606** 为例：

```json
{
    "symbol": "v2606",
    "exchange": "DCE",
    "name": "聚氯乙烯期货",
    "size": 5,
    "price_tick": 5,
    "min_volume": 1
}
```

**操作步骤**：
1. 打开 `tqsdk_contracts.json` 文件
2. 找到合适的插入位置（建议按品种和到期月份排序）
3. 在合适的位置插入上述 JSON 对象
4. 确保 JSON 格式正确（逗号分隔）

### 步骤 3：添加期权合约

以添加 **PVC期权 v2606** 为例，需要添加多个行权价的看涨和看跌期权：

```json
{
    "symbol": "v2606C5500",
    "exchange": "DCE",
    "name": "PVC期权看涨",
    "size": 5,
    "price_tick": 5,
    "min_volume": 1
},
{
    "symbol": "v2606P5500",
    "exchange": "DCE",
    "name": "PVC期权看跌",
    "size": 5,
    "price_tick": 5,
    "min_volume": 1
},
{
    "symbol": "v2606C6000",
    "exchange": "DCE",
    "name": "PVC期权看涨",
    "size": 5,
    "price_tick": 5,
    "min_volume": 1
},
{
    "symbol": "v2606P6000",
    "exchange": "DCE",
    "name": "PVC期权看跌",
    "size": 5,
    "price_tick": 5,
    "min_volume": 1
}
```

**注意事项**：
1. 期权合约必须成对添加（看涨和看跌）
2. 每个行权价都需要添加看涨和看跌两个合约
3. 建议添加多个行权价以形成完整的期权链
4. 期权合约的 `size` 和 `price_tick` 通常与期货合约相同

### 步骤 4：验证配置

添加完成后，使用验证脚本检查配置：

```bash
python vnpy/gateway/tqsdk/simple_verify.py
```

该脚本会：
- 加载配置文件
- 统计合约数量
- 列出所有期权合约
- 验证期权链完整性

## 期权链配置建议

为了构建完整的期权链，建议：

1. **行权价数量**：每个到期日至少添加 5-10 个行权价
2. **行权价间隔**：根据品种特性选择合适的间隔（如 50、100、250 等）
3. **覆盖范围**：行权价应覆盖平值、实值和虚值区间
4. **成对添加**：每个行权价必须同时添加看涨和看跌期权

**示例：PVC期权链配置**

假设当前 PVC 期货价格为 6500，建议配置以下行权价：

```
5500  - 深度虚值
5750  - 虚值
6000  - 虚值
6250  - 轻度虚值
6500  - 平值
6750  - 轻度实值
7000  - 实值
7250  - 实值
7500  - 深度实值
```

## 注意事项

### 1. JSON 格式

- 确保所有字符串使用双引号 `"`
- 数值不要使用引号
- 对象之间用逗号 `,` 分隔
- 数组最后一个元素后面不要有逗号

### 2. 合约代码大小写

- **DCE/SHFE**：品种代码通常使用小写（如 `v2605`, `cu2605`）
- **CZCE**：品种代码通常使用大写（如 `SR605`, `CF605`）
- **CFFEX**：品种代码使用大写（如 `IF2605`, `IO2605`）

### 3. 期权合约命名

- **CFFEX 股指期权**：使用 `-` 分隔（如 `IO2605-C-4500`）
- **DCE/SHFE 商品期权**：不使用分隔符（如 `v2605C6000`）
- **CZCE 商品期权**：不使用分隔符（如 `SR605C6000`）

### 4. 合约乘数和价格变动

这些参数必须与交易所规则一致，否则可能导致：
- 下单失败
- 价格计算错误
- 风险控制不准确

### 5. 配置文件更新

修改配置文件后：
1. 重启天勤网关
2. 检查日志确认合约加载成功
3. 在期权模块中验证合约可见性

## 故障排查

### 问题 1：合约加载失败

**症状**：网关启动后日志显示合约数量为 0 或不正确

**解决方案**：
1. 检查 JSON 格式是否正确
2. 使用 JSON 验证工具验证文件
3. 检查文件路径是否正确

### 问题 2：期权合约无法识别

**症状**：期权模块中看不到期权合约

**解决方案**：
1. 确认期权合约命名规则正确
2. 检查交易所代码是否正确
3. 查看网关日志是否有解析错误

### 问题 3：期权链不完整

**症状**：T型报价中缺少部分期权

**解决方案**：
1. 确保每个行权价都有看涨和看跌期权
2. 检查行权价是否连续
3. 运行验证脚本检查完整性
